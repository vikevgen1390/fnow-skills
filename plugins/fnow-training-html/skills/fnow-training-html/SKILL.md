# Forecast NOW! — HTML-конспект обучения из транскрипта

## Обзор

Скилл преобразует сырой транскрипт обучения (TXT с таймкодами и спикерами) в самодостаточный HTML-конспект в стиле fnow.ru. Включает ссылки на документацию help.fnow.ru и подходящие видео из каталога Rutube. В конце — оценка судьёй Opus с опциональным repair loop.

**REQUIRED:** `mcp__kb-gateway__kb_search` — для заземления ответов Q&A и поиска ссылок.

**Benchmark (4 обучения, изолированный прогон):** mean 91.1/100 · weakest: links_quality · strongest: design_system.

## Входные данные

- Путь к файлу транскрипта (`.txt`, формат «Спикер N: текст»)
- Номер и название обучения (если не указаны — извлечь из имени файла или первого блока)
- Путь для сохранения HTML (по умолчанию — та же папка, что и транскрипт)
- Каталог видео: `C:\Users\e.viktorov\Desktop\AI\video\catalog_rutube_fnow_detailed.md`

## Пайплайн (8 шагов)

### Шаг 1 — Чтение транскрипта

Прочитай файл целиком. Зафиксируй:
- Дату, название, номер обучения, компания-клиент
- **Полный хронологический список тем** — всё, о чём говорил ведущий, включая короткие демо. Нумеруй — понадобится для шага 2d.

### Шаг 2a — Извлечение структуры (без формулировки ответов)

**A. Темы плана** (хронологически, рабочие названия — ответы не писать)

**B. Вопросы участников** (только реальные из транскрипта, 6–10 штук)

**C. Ситуации клиента** (только реальные из транскрипта, 2–4 штуки)

**D. Темы теста** — 10 концепций, по одной на каждую тему плана

### Шаг 2b — KB Search (до формулировки ответов)

Для каждого вопроса из 2a-B и каждой темы плана — выполни `mcp__kb-gateway__kb_search`.

**Правило заземления:** формула / порог / число без KB и без прямой цитаты ведущего → не включай. Пиши «по словам ведущего» или пропусти.

### Шаг 2c — Формулировка

**Q&A:** ответ = KB-результат + слова ведущего. Никаких синтезированных формул.

**Кейсы:** только реальные ситуации из транскрипта. Синтетические кейсы запрещены.

**Тест (10 вопросов):**
- 4 варианта, 1 правильный
- Составь список позиций правильного ответа перед записью
- Каждая буква А/Б/В/Г — минимум 2 раза из 10. Если нет — перетасуй варианты.

### Шаг 2d — Self-Audit (до генерации HTML)

```
[ ] Coverage: темы шага 1 vs темы плана — пропуски?
    → добавить или явно обосновать исключение

[ ] Test distribution: А/Б/В/Г каждая ≥ 2 раза из 10
    → если нет — перетасовать варианты в нужных вопросах

[ ] Answer grounding: формулы/числа без KB и без слов ведущего?
    → удалить или заменить на «по словам ведущего»

[ ] Links: все ссылки из справочника? Ни одного generic /параметры без #якоря?
    → исправить до шага 2e
```

### Шаг 2e — Валидация ссылок

1. Все URL — из справочника ниже. Справочник верифицирован через WebFetch — не нужно перепроверять.
2. Для URL вне справочника — `WebFetch(url, "NOT FOUND?")`. NOT FOUND → заменить родительским из справочника.
3. Нет валидного URL → не ставить ссылку (лучше без, чем нерабочая).

### Шаг 2f — Подбор видео из каталога

Прочитай `C:\Users\e.viktorov\Desktop\AI\video\catalog_rutube_fnow_detailed.md`.

Для каждой план-плитки найди видео, которое **ПРЯМО** раскрывает её тему:

**Правило релевантности — СТРОГОЕ:**
- Добавляй только если видео прямо объясняет или демонстрирует тему плитки
- Максимум 2 видео на плитку
- **Лучше без видео, чем нерелевантное** — не натягивай
- Если прямой связи нет — оставь плитку без видео

**Типы видео** (для badge):
| Тип | badge | Признак |
|---|---|---|
| Урок | `lesson` | Скринкаст, Кластер 10, обычно <20 мин |
| Вебинар | `webinar` | Запись вебинара, >30 мин |
| Воркшоп | `workshop` | Практический воркшоп с разбором |

**HTML для видеоблока** (добавляется ПОСЛЕ `.plan-links`, ВНУТРИ план-плитки):
```html
<div class="plan-videos">
  <div class="plan-videos-label">▶ Видео по теме</div>
  <a class="plan-video-link" href="https://rutube.ru/video/VIDEO_ID/" target="_blank">
    <span class="video-badge lesson">Урок</span>
    Название видео из каталога
    <span class="video-dur">12:43</span>
  </a>
</div>
```

### Шаг 3 — Генерация HTML

**Структура страницы:**
1. Hero — номер, название, дата, компания, теги-темы
2. `section-white` (#fff) — «01 — План обучения» — плитки 2×N
3. `section-gray` (#F5F7F9) — «02 — Вопросы и ответы» — плитки 2×N
4. `section-blue` (#003366) — «03 — Кейсы» — плитки 2×N
5. `section-lightblue` (#E6F6FF) — «04 — Тест» — плитки 2×N + JS
6. Footer — название, дата, ссылки help.fnow.ru, fnow.ru

**ОТЧУЖДАЕМОСТЬ — ОБЯЗАТЕЛЬНО. HTML должен работать на любой машине:**
```
❌  <link href="file:///C:/Users/...">     ← ЗАПРЕЩЕНО
❌  url('file:///C:/Users/...')            ← ЗАПРЕЩЕНО
✅  Hero: только CSS-градиент, без url()
✅  Шрифты: CSS-стек без <link>
```

**CSS-шаблон (весь необходимый):**
```css
:root {
  --brand:#0066CA; --brand-dark:#003366;
  --red:#FF6565; --green:#059669;
  --text:#000; --text-body:#545454; --text-gray:#5F5F5F;
  --bg-gray:#F5F7F9; --bg-blue:#E6F6FF; --bg-white:#fff;
  --border:#D9D9D9; --border-light:#EEE;
  --font:'Outfit','Helvetica Neue',Arial,sans-serif;
}

/* Hero — ТОЛЬКО градиент, без url() */
.hero {
  background: linear-gradient(150deg,rgba(0,51,102,.93) 0%,rgba(0,102,202,.84) 55%,rgba(46,139,224,.78) 100%);
  padding: 80px 0 90px;
}

/* Кейсы на тёмной секции — НЕЛЬЗЯ #fff */
.case-tile {
  background: rgba(255,255,255,.06);
  border: 1px solid rgba(255,255,255,.12);
}

/* Видео в план-плитках */
.plan-videos{border-top:1px solid var(--border-light);margin-top:12px;padding-top:10px;display:flex;flex-direction:column;gap:5px}
.plan-videos-label{font-size:11px;font-weight:700;letter-spacing:.06em;text-transform:uppercase;color:var(--text-gray);margin-bottom:2px}

.plan-video-link:hover{border-color:var(--brand);color:var(--brand);text-decoration:none}
.video-badge{font-size:10px;font-weight:700;padding:2px 6px;flex-shrink:0;letter-spacing:.04em;text-transform:uppercase}
.video-badge.lesson{background:rgba(0,102,202,.08);color:var(--brand);border:1px solid rgba(0,102,202,.2)}
.video-badge.webinar{background:rgba(245,158,11,.08);color:#92400e;border:1px solid rgba(245,158,11,.3)}
```

**Прочие CSS-правила:**
- `border-radius: 0` везде — строго (правило бренда)
- Плитки плана: hover → `border-color: var(--brand)`
- Тест: интерактивный JS — клик → правильный (зелёный) / неправильный (красный), финальный балл, кнопка «Пройти заново»

Файл сохраняется в ту же папку, что и транскрипт. Имя: `обучение-N.html`.

### Шаг 4 — Судья (Opus)

Запусти `Agent` с `model: "opus"`. Передай:
- Хронологический список тем (шаг 1)
- Перечень Q&A, кейсов, тем теста из HTML
- Промпт из `prompt-judge.md`

Судья возвращает структурированный отчёт — выведи пользователю перед финальным сообщением.

### Шаг 5 — Repair Loop (условный)

**Условие:** хотя бы один критерий судьи < 70/100.

1. Прочитай критические замечания
2. Точечные правки (не полная регенерация):
   - Тест: только перетасовать варианты
   - Q&A: только исправить неточные ответы
   - Кейсы: заменить синтетический на реальный из транскрипта
   - Видео: убрать нерелевантное (лучше без)
3. Повтори шаг 4 (судья v2)
4. **Cap = 1 итерация.** Если после repair всё ещё < 70 → предупредить и остановиться.

| Блок HTML | Источник | Шаг |
|---|---|---|
| Plan tiles + видео | Ведущий из транскрипта + каталог видео | 2a→2d, 2f |

## Частые ошибки

| Ошибка | Правило |
|---|---|
| `<link href="file:///...">` | **ЗАПРЕЩЕНО** — нарушает отчуждаемость |
| `url('file:///...')` в hero | **ЗАПРЕЩЕНО** — только CSS-градиент |
| Белый текст невидим в кейсах | `.case-tile { background: rgba(255,255,255,.06) }` — не `#fff` |
| Правильные ответы только Б и В | Self-audit шаг 2d: А/Б/В/Г каждая ≥ 2 раза из 10 |
| Формула/число без источника | KB search до ответа (шаг 2b); иначе — «по словам ведущего» |
| Синтетический кейс | Только реальные ситуации из транскрипта |
| Нерелевантное видео | Лучше без, чем нерелевантное. Прямая связь с темой — обязательна |
| Пропуск темы | Coverage check шаг 2d |
| `border-radius > 0` | Строго 0 |
| Generic `/параметры` без якоря | Использовать `#конкретный_якорь` из справочника |
| `/автопилот` | **УСТАРЕВШАЯ** — заменить на `/фильтры_автоподготовки_заказов` |
| `/балансировка` | Заменить на `/балансировка_запасов#якорь` |
| `/сезонность` | Заменить на `/параметры/настройка_параметров#месячная_сезонность` |
| `/уровень_сервиса` | Заменить на `/параметры/настройка_параметров#уровень_сервиса` |
| `/оптимальный_уровень_сервиса` | Заменить на `/анализ/оптимизатор_уровня_сервиса` |
| `/иерархия_складов` | Заменить на `/параметры/настройка_параметров#иерархия_складов` |
| `/abc_xyz_анализ` | Заменить на `/анализ/abc-xyz` |
| `/правила_автоустановки_параметров` | Заменить на `/параметры/настройка_параметров#правила_установки_параметров` |

# Forecast NOW! — HTML-конспект обучения из транскрипта

## Обзор

Скилл преобразует сырой транскрипт обучения (TXT с таймкодами и спикерами) в самодостаточный HTML-конспект в стиле fnow.ru. В конце — автоматическая оценка судьёй на модели Opus с опциональным repair loop.

**REQUIRED SUB-SKILL:** Используй `fnow-skills:fnow-design` для генерации HTML.
**REQUIRED SUB-SKILL:** Используй `mcp__kb-gateway__kb_search` для заземления ответов Q&A и ссылок.

## Входные данные

- Путь к файлу транскрипта (`.txt`, формат «Спикер N: текст»)
- Номер и название обучения (если не указаны — извлечь из имени файла или первого блока)
- Путь для сохранения HTML (по умолчанию — та же папка, что и транскрипт)

## Пайплайн (7 шагов)

### Шаг 1 — Чтение транскрипта

Прочитай файл целиком. Зафиксируй:
- Дату, название, номер обучения, компания-клиент
- Имя ведущего (специалист Forecast NOW!)
- **Полный хронологический список тем** — всё, о чём говорил ведущий, включая короткие демо и вступительные блоки. Нумеруй список — он понадобится для проверки на шаге 2d.

### Шаг 2a — Извлечение структуры (без формулировки ответов)

Из транскрипта извлеки заготовки — **не пиши финальные ответы ещё**:

**A. Темы плана** (хронологически):
- Для каждой темы: краткое рабочее название
- Включай даже короткие демо-блоки — исключать тему можно только с явным обоснованием

**B. Вопросы (только вопросы, без ответов)**:
- Только реальные вопросы участников из транскрипта
- Минимум 6, максимум 10

**C. Ситуации клиента** (только описание проблемы, без рекомендации):
- Реальные затруднения, описанные клиентом
- Минимум 2, максимум 4

**D. Темы теста** — список из 10 концепций, по одной на каждую тему плана

### Шаг 2b — KB Search (до формулировки ответов)

**Выполни kb_search для каждого вопроса из шага 2a-B** — это основной источник для ответов.

Также выполни kb_search по ключевым словам каждой темы плана — для ссылок help.fnow.ru.

**Правило заземления:** если утверждение (особенно формула, порог, число) не найдено в KB и не сказано дословно ведущим — не включай его в ответ. Напиши «по словам ведущего на обучении» или пропусти деталь.

### Шаг 2c — Формулировка ответов и рекомендаций

Теперь формулируй:

**Q&A:** для каждого вопроса — ответ на основе KB-результатов + слов ведущего из транскрипта. Никаких синтезированных формул без источника.

**Рекомендации к кейсам** — конкретные параметры и настройки, подтверждённые транскриптом или KB.

**Вопросы теста** (10 штук):
- По одному на каждую тему плана
- 4 варианта ответа, ровно один правильный
- **ОБЯЗАТЕЛЬНО:** составь список позиций правильного ответа. Каждая буква А/Б/В/Г должна встречаться минимум 2 раза из 10. Если А или Г не встречаются — перетасуй варианты ответов в соответствующих вопросах.

### Шаг 2d — Self-Audit Checklist (выполнить до генерации HTML)

Пройди по каждому пункту и зафикисруй результат явно:

```
[ ] Coverage: темы из хронологического списка (шаг 1) vs темы плана — есть пропуски?
    → Если да: добавить в план ИЛИ явно написать «исключена, т.к. [причина]»

[ ] Test distribution: посчитай А/Б/В/Г среди правильных ответов
    → Цель: каждая позиция ≥ 2 раза из 10
    → Если нет — перетасовать варианты в вопросах, где позиция избыточна

[ ] Answer grounding: есть ли в Q&A формулы/числа/пороги не из транскрипта и не из KB?
    → Если да — удалить или заменить на «по словам ведущего»

[ ] Links: для каждой темы плана — найден ли KB-результат, из которого взят URL?
    → Строить URL вида https://help.fnow.ru/раздел только на основе реальных KB-названий
```

Если любой пункт выявил проблему — исправь до перехода к шагу 2e.

### Шаг 2e — Валидация ссылок (WebFetch)

Для каждой ссылки, которую планируешь поставить в план-плитки:

1. **Определи целевой URL** из справочника ниже (раздел «Проверенные URL help.fnow.ru»).
   - Если тема есть в справочнике → используй URL из него, не придумывай.
   - Если темы нет в справочнике → используй `WebFetch` для проверки (шаг 3).

2. **Для URL, которых нет в справочнике**, выполни проверку:
   ```
   WebFetch(url, "Верни заголовок страницы и первые 3 раздела. Если страница не найдена (404), напиши 'NOT FOUND'.")
   ```
   - Ответ содержит «NOT FOUND» → URL недействителен → замени на ближайший родительский из справочника.
   - Если URL с якорем `#section` — проверь, упоминается ли этот якорь в ответе.

3. **Пустые или сгенерированные ссылки запрещены.** Если валидный URL не найден — не ставь ссылку совсем (лучше без ссылки, чем с нерабочей).

4. Финальная таблица ссылок перед генерацией HTML:

   | Тема плана | URL | Статус |
   |---|---|---|
   | ... | `https://help.fnow.ru/...` | ✓ / FIXED / NO LINK |

**Не нужно проверять** URL из справочника ниже — они уже верифицированы.

### Шаг 3 — Генерация HTML

Применяй **fnow.ru Landing стиль** из `fnow-skills:fnow-design`.

**Обязательная структура:**
1. **Hero** — название, номер, дата, компания, теги-темы
2. **Section white** — «01 — План обучения» — плитки 2×N
3. **Section gray** — «02 — Вопросы и ответы» — плитки 2×N
4. **Section dark** (`background: #003366`) — «03 — Кейсы» — плитки 2×N
5. **Section lightblue** — «04 — Тест» — плитки 2×N + интерактивный JS
6. **Footer** — название, дата, ссылки на help.fnow.ru и fnow.ru

**Критичное CSS-правило для кейсов (тёмная секция):**
```css
/* Тёмный фон секции — плитки НЕ могут быть белыми, иначе белый текст невидим */
.case-tile {
  background: rgba(255,255,255,.06);  /* НЕ #fff */
  border: 1px solid rgba(255,255,255,.12);
}
```

**Остальные CSS-правила:**
- `border-radius: 0` везде — строго
- Шрифт Outfit (через fonts-local.css)
- Цвета: `--brand:#0066CA`, `--brand-dark:#003366`
- Все плитки — `border: 1px solid var(--border)`, hover — `border-color: var(--brand)`
- Тест — интерактивный JavaScript: клик → правильный/неправильный, финальный балл

Файл сохраняется в ту же папку, что и транскрипт. Имя: `обучение-N.html`.

### Шаг 4 — Судья (Opus)

После генерации HTML запусти `Agent` с `model: "opus"`, передав:
- Хронологический список тем из транскрипта (шаг 1)
- Перечень Q&A, кейсов, тем теста из HTML
- Промпт из файла `prompt-judge.md` этого скилла

Судья возвращает структурированный отчёт. **Выведи его пользователю** перед финальным сообщением.

### Шаг 5 — Repair Loop (условный)

**Условие запуска:** судья выставил < 70/100 хотя бы по одному критерию.

**Если запускаем:**
1. Прочитай критические замечания судьи
2. Примени точечные исправления к соответствующим разделам HTML (не полная регенерация)
   - Тест: только перетасовать варианты в проблемных вопросах
   - Q&A: только исправить неточные ответы
   - План: только добавить пропущенные темы
3. Повтори шаг 4 (судья v2)
4. **Cap = 1:** если после repair хотя бы один критерий всё ещё < 70 — вывести предупреждение пользователю и остановиться

**Если не запускаем (все критерии ≥ 70):** сообщить итоговый балл и завершить.

## Быстрая справка

| Блок HTML | Источник данных | Шаг |
|---|---|---|
| Plan tiles | Ведущий из транскрипта | 2a→2d |
| Q&A tiles | Вопросы участников + KB | 2b→2c |
| Case tiles | Ситуации клиента + KB | 2b→2c |
| Test tiles | Темы плана | 2c→2d |

## Частые ошибки

| Ошибка | Правило |
|---|---|
| Белый текст невидим в кейсах | `.case-tile { background: rgba(255,255,255,.06) }` — никогда не `#fff` в тёмной секции |
| Правильные ответы только Б и В | Self-audit шаг 2d: А/Б/В/Г каждая ≥ 2 раза |
| Формула без источника в Q&A | KB search до формулировки (шаг 2b); без KB-подтверждения — «по словам ведущего» |
| Пропуск темы из транскрипта | Coverage check шаг 2d: список тем шаг 1 vs темы плана |
| border-radius > 0 | Строго 0, это правило бренда fnow.ru |
| Внешние CDN-зависимости | Только локальные шрифты из fonts-local.css |
| Выдуманные ссылки help.fnow.ru | Использовать только URL из справочника ниже или проверять WebFetch (шаг 2e) |
| Ссылка `/параметры` без якоря | Использовать `/параметры/настройка_параметров#конкретный_якорь` |
| Ссылка `/балансировка` | Использовать `/балансировка_запасов#конкретный_якорь` |
| Ссылка `/уровень_сервиса` | Использовать `/параметры/настройка_параметров#уровень_сервиса` |
| Ссылка `/иерархия_складов` | Использовать `/параметры/настройка_параметров#иерархия_складов` |
| Ссылка `/сезонность` | Использовать `/параметры/настройка_параметров#месячная_сезонность` |
| Ссылка `/прогнозирование` | Использовать `/спрос` или `/параметры/настройка_параметров#порог_дефицита` |
| Ссылка `/оптимальный_уровень_сервиса` | Использовать `/анализ/оптимизатор_уровня_сервиса` |
| Ссылка `/правила_автоустановки_параметров` | Использовать `/параметры/настройка_параметров#правила_установки_параметров` |
| Ссылка `/abc_xyz_анализ` | Использовать `/анализ/abc-xyz` |
| Ссылка `/консольные_команды` | Использовать `/консоль` |
| Ссылка `/пользователи` | Использовать `/настройка_прав_пользователей` |
| Ссылка `/автопилот` | **УСТАРЕВШАЯ** — использовать `/фильтры_автоподготовки_заказов` или `/формирование_заказа#автоматическое_формирование_заказа` |
| Ссылка `/параметры/настройка_параметров` без якоря (когда тема — округление) | Использовать `#округление` или `#кратность` или `#минимальная_партия_ед` |
| Ссылка на «Параметры» когда тема — календари | Использовать `/настройки/календари#календарь_продаж` и `#календарь_заказов` |
